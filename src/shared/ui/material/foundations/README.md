# Material foundations

This directory mirrors the official Material `Foundations` navigation.

Create a foundation domain only when current component or product work requires a real cross-component contract. Use the official documentation slug.

Examples of eligible domains include accessibility, adaptive/layout, and interaction foundations.

Each implemented domain documents its current contract, omitted capability, known issues, consumers, verification, and review status in a local `README.md`. An independent review may add a colocated `AUDIT.md`.

Do not create placeholder runtime directories for every documentation page.
