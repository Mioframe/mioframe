# Material component contract

Run inside the `material-component` orchestrator context. Correction selection shares invocation scope, canonicalization preflight, accepted evidence, dependency graph, prerequisite stack, and continuation state.

## Inputs

Receive:

- family, invocation scope, mode, objective, scenarios, and non-goals;
- current canonicalization preflight, orientation, and normalized workflow state;
- concern plan;
- actual required dependency inventory for the supported surface;
- required target slices and selected specialist results;
- current owners, public exports, representative consumers, legacy-owner state, and proof.

Block when invocation scope is missing, persisted state was not reconciled with code, a required lane is missing, dependency ownership is unknown, or production work preceded the gate.

## Responsibility

Synthesize:

- applicable target and source decisions;
- current ownership and canonicalization state;
- complete dependency closure and exact prerequisites;
- durable public, token, style, and motion facts;
- highest-priority complete correction unit;
- proof lane, failing observation, compatibility/visual impact, operator requirement, and completion condition;
- remaining family gaps and exact external blockers.

Do not repeat accepted research. Challenge contradictions, missing ownership, incomplete imports, unsupported generalizations, and missing proof.

## Invocation scope

A family-only request is `full-family`. A persisted objective, roadmap next action, or previous bounded unit cannot change it to `focused-correction`.

For `full-family`, the contract must plan continued convergence after the current unit. It must not describe remaining required work as later independent passes or future operator tracks.

## Dependency closure

For every dependency required by the supported surface record:

```text
DEPENDENCY
Contract:
Current owner:
Classification: canonical-foundation | canonical-family | temporary-legacy-material | project-extension | generic-foundation
Readiness: ready | defective | missing | parallel-owner
Required proof:
Replacement obligation: none | foundation prerequisite | official-family prerequisite | remove/defer extension
Orchestration status: ready | execute-now | externally-blocked
```

A dependency used by the family remains in the calling orchestration regardless of implementation owner.

A temporary legacy or missing family-agnostic contract becomes an exact `material-foundation` prerequisite. Another official component requires its owning family public contract. These are `execute-now`, not terminal blockers or out-of-scope work.

Select the smallest coherent required prerequisite and track cycles.

## Canonicalization triggers

Set `Canonicalization trigger: yes` when current or proposed state creates/preserves a candidate canonical owner, changes the Material root export, migrates consumers, removes/forwards the legacy owner, or claims canonical/adoption/alignment status.

When triggered:

- dependency inventory covers the complete supported surface, not only edited files;
- all required dependencies must be ready before export/adoption/removal can pass;
- an already-premature export/adoption makes dependency closure or safe rollback the highest-priority correction;
- no actually used dependency may be classified outside the correction.


Wrong dependency/foundation ownership outranks local semantics, token cleanup, geometry, and motion for the same supported surface.

Implementation cannot proceed while a required prerequisite is pending. A lower-priority local correction cannot bypass blocked closure.

## Documentation

Write current truth only: invocation scope, preflight outcome, target decisions, ownership, dependency closure, prerequisite stack, durable contract, proof obligation, current correction, and remaining gaps.

Remove stale objectives, review history, round narratives, shell transcripts, route inventories, scorecards, and future-pass descriptions.

Set workflow stage to `contract-review`. Return control to the orchestrator.

## Exit gate

Pass only when:

- current invocation scope and preflight are valid;
- every required selected lane is complete;
- actual dependencies have explicit owner/readiness/orchestration status;
- canonicalization triggers widened closure correctly;
- the correction is highest priority;
- prerequisites are exact and executable;
- proof and compatibility decisions are locked;
- remaining internal work requires continuation rather than termination.

## Forbidden

- direct user invocation;
- production, proof, story, or consumer edits;
- trusting stale README scope or dependency conclusions;
- treating a used dependency as outside orchestration;
- accepting open closure for canonical export, adoption, legacy removal, or alignment;
- moving another component family into foundation;
- choosing lower-priority local work around ownership blockers;
- describing internal prerequisites as future operator work;
- roadmap advancement, Git operations, durable histories, registries, checklists, scorecards, or progress ledgers.
