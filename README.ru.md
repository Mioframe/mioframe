# Beaver

Локальный менеджер персональных данных

[Open App](https://vyachean.github.io/beaver/)

## О проекте

Beaver - это локально ориентированное приложение для управления персональными данными, разработанное с использованием современных веб-технологий. Оно предоставляет пользователям возможность хранить, структурировать и управлять своими данными без необходимости использовать облачное хранилище. Приложение использует технологию Origin Private File System (OPFS) для локального хранения данных в браузере.

## Ключевые особенности

- **Локальное хранение данных**: Все данные хранятся локально в браузере пользователя
- **Поддержка различных типов данных**: Строки, числа, логические значения, дата, связь с другими записями
- **Гибкая структура данных**: Возможность создавать пользовательские свойства и представления
- **Интуитивный интерфейс**: Простое и удобное управление данными
- **Поддержка различных типов хранилищ**: Включает поддержку OPFS и потенциально других провайдеров

## Начало работы

### Требования

- Node.js
- pnpm 10

### Установка

1. Клонируйте репозиторий:

```bash
git clone https://github.com/Vyachean/self-base.git
```

2. Установите зависимости:

```bash
pnpm install
```

### Разработка

```bash
pnpm dev
```

Запускает локальный HTTPS dev-сервер на Vite.

### Сборка для продакшена

```bash
pnpm build
pnpm preview
```

### Запуск тестов

```bash
# Vitest в watch-режиме
pnpm test

# Vitest одним прогоном
pnpm test:run

# Vitest с покрытием
pnpm test:coverage

# StrykerJS mutation testing
pnpm test:mutate

# Dry-run конфигурации StrykerJS
pnpm exec stryker run --dryRunOnly

# При необходимости сузить область мутаций прямо из терминала
pnpm exec stryker run -m "src/shared/lib/**/*.ts"
```

Browser smoke и end-to-end сценарии запускаются через Playwright:

```bash
# Один раз на машину установить browser-бинарь Playwright
pnpm e2e:install

# Headless browser e2e
pnpm e2e

# UI runner Playwright
pnpm e2e:ui

# Headed-режим для локальной отладки
pnpm e2e:headed
```

`Vitest` используйте для unit и integration проверок внутренней логики, сервисов, VFS, адаптеров и composables. `Playwright` оставлен только для browser smoke и e2e-сценариев через пользовательский UI. `pnpm test:mutate` запускает StrykerJS с динамическим `stryker.config.mjs`, который выводит список файлов для мутаций по colocated `*.test.ts`, исключает `src/shared/ui` и при необходимости дополнительно сужается CLI-флагами вроде `-m`.

## Линтинг и форматирование

В проекте используются `oxlint` и `eslint` для линтинга, а `oxfmt` для форматирования. Доступны такие команды:

```bash
# Полный lint-пайплайн
pnpm lint

# Только oxlint
pnpm lint:oxlint

# Только eslint
pnpm lint:eslint

# Форматирование через oxfmt
pnpm format
```

Для точечных исправлений по изменённым файлам предпочитайте прямые команды вроде `pnpm exec eslint --fix <path ...>`, `pnpm exec oxlint <path ...>`, и `pnpm exec oxfmt <path ...>`.
